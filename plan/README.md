## 実装依頼の出し方

- `03-a を実装して` のように ID を指定 → その項目だけ実装
- `P0 を全部実装して` → 全チケット横断で優先度P0のものを一括実装
- `03 を全部実装して` → チケット03の全項目を優先度順に実装
- 単に `続きをやって` → 未着手のうち最優先（下表の最上位）から着手

## 共通点（横断観察）

- 01・02は共通して「本質的でない手間・注意力の分散を取り除く」という動機
  （読書に集中したい／起動作業に気を取られたくない）が背景にある。
- 03は単一の根本原因（`tei_parse.py` が GROBID TEI の `teiHeader`/`back`/
  `graphic` を一切読まず `text/body` だけを見ている）から6項目が派生しており、
  個別に直すのではなく「teiHeaderとbackを読む処理を新設する」1本の作業で
  まとめて解消する設計にしている（詳細は03ファイル内「診断の共通根」参照）。
- 01（図表パネルの独立スクロール枠）と03-c（参考文献パネル）は同じUI設計
  （本文とは別に独立スクロールする側パネル、非アクティブ時は減光）を流用できる。
  03-eのTOCも同系統のUI部品になり得るため、実装時にこの3つで共通コンポーネント
  化を検討する余地がある。
- 04-eは03-cの副産物として一部自然に解消する見込み（参考文献が`text/back`に
  正しく分離されれば、本文の語彙抽出にも混入しなくなる）。03を先に実装した
  ほうが04の手戻りが減る可能性がある。
- 03の反省（未検証の推測でロジックを設計しない）は04にも適用した: 04-aは
  「複合語は個々の単語が一般語でも専門用語になり得る」という誤判定リスクを
  踏まえ単語1語の候補のみに絞り、04-eは03-cの効果を実データで見てから
  追加対応を判断する設計にしている。

## 自己検証インフラ（Docker/GROBID/ネットワーク不要）

実際にGROBIDを起動しなくても、上記チケットの実装を安く検証できるよう
以下を整備済み（`tests/`配下、本番コードは触っていない）:

- `tests/fixtures/sample_fulltext.tei.xml` — teiHeader（タイトル・著者2名・
  要旨）、`<ref type="bibr">`引用、`back/listBibl`参考文献（DOI付き/無し各1件）、
  ネストしたdiv（番号なし見出しを含む）、`<graphic coords="...">`付きの図を
  1つの小さな文書にまとめた、手作りの「模擬GROBID出力」。GROBID公式ドキュメント
  ・実出力例をもとにした最善努力の再現であり、実際のライブGROBID出力の
  確認ではない旨をファイル冒頭のコメントに明記している。課題02のDocker環境が
  整った際に実データと突き合わせて精度を検証すべきタスクとして残っている。
- `tests/helpers.py` — `blank_pdf()`（座標クロップ用の合成PDFをその場生成、
  実物PDFを同梱しない）と`load_golden_tei()`。
- `tests/conftest.py` — `isolated_data_dir`フィクスチャ。`storage.DATA_DIR`等が
  リポジトリ直下`data/`に決め打ちのため、明示的に要求しないとパイプライン統合
  テストが本物の`data/daily-read.db`を汚してしまう。これを`tmp_path`に
  差し替える。
- `tests/test_pipeline_golden_fixture.py` — 上記フィクスチャで
  `app.pipeline.process_upload`をGROBID呼び出しごとモックしてエンドツーエンドに
  実行し、**現状の（未修正の）挙動をそのままアサーションとして固定**した
  回帰テスト。各アサーションに対応するチケットIDをコメントで明記しており、
  該当チケットを実装したら期待値をそのアサーションだけ更新する運用とする。
  `pytest tests/`で1秒未満・完全オフラインで実行できる。

このフィクスチャを実際に動かして検証した過程で、**新たな不具合を1件発見した**
（[04-g](04-glossary-quality-and-user-vocabulary.md#04-g-見出し直後の本文定義が検出されない回帰新規発見):
見出し直後に本文定義が来ると検出が壊れる）。これは狙い通りの効果で、
今後もチケット実装時にまずこのフィクスチャで試すことを標準の確認手順とする。
